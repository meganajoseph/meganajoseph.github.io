Work Experience

Consultant at Control Plane

Duration: May 2024 – August 2024

Role Overview:

During my time as a consultant at Control Plane, I was tasked with developing an AI-driven chatbot to enhance customer interactions with the company’s platform. The project was a comprehensive exercise in both innovation and adaptability, requiring me to explore various approaches to create a robust solution.

Initial Development: Retrieval-Augmented Generation (RAG) Framework

I began by building a Retrieval-Augmented Generation (RAG) framework using the Llama 3 8B model. This framework aimed to leverage existing company documentation to provide accurate and contextually relevant responses through the chatbot. The process involved:

  • Data Processing: I employed FireCrawl to systematically extract and convert the entire documentation site into markdown format.
  • Semantic Chunking & Embedding: Using LangChain’s semantic chunker, I segmented the documents into manageable chunks. These chunks were then vectorized with Nomic-embed-text and stored in ChromaDB.
  • Visualization: To facilitate user interaction and debugging, I integrated Gradio for real-time chatbot visualization.

The initial implementation of the RAG framework performed well, demonstrating the potential of leveraging documentation for dynamic chatbot responses.

Exploration of Fine-Tuning Techniques

Encouraged by the initial success, I explored fine-tuning the Llama models to improve the chatbot’s performance using Unsloth. I used LoRA fine-tuning on both Llama 3 8B and Llama 3.1 8B models, training them on a custom Q&A dataset derived from the company’s documentation. However, despite rigorous tuning, the models continued to exhibit hallucination tendencies, which are common challenges in large language models. This led to a strategic decision to pivot back to a RAG-based approach, but this time with a focus on GraphRAG, a method designed to enhance contextual accuracy through graph-based retrieval.

GraphRAG Development

I initiated the development of a GraphRAG framework, which aimed to integrate graph structures into the RAG architecture, allowing the chatbot to understand and respond with greater contextual relevance. This approach promised to mitigate the hallucination issues by grounding responses more effectively in structured data. Unfortunately, due to budget constraints, Control Plane had to end my contract before the GraphRAG framework could be fully realized. However, the work offered valuable insights into the complexities and potential of AI-driven chatbots.

Takeaways

Working on this project was a highly educational experience. It not only honed my technical skills across a variety of tools and techniques but also underscored the importance of flexibility and iterative problem-solving in AI development. The knowledge and experience I gained were invaluable and continue to inform my approach to AI projects.

Projects

Pololu Lane Keeping

Pololu Lane Keeping is a project from an embedded systems class I took. We worked on a robot car called Pololu and enabled it to detect lanes, drive, and perform lane changes. It uses a camera, a Raspberry Pi 4, and OpenCV to achieve this functionality. The Raspberry Pi was connected to the Pololu via UART.

To detect lanes, we applied a Gaussian filter to eliminate noise and a binary filter to process the images from the camera. As this was an embedded systems project, we developed different modes for the Pololu based on the lane positions received from the Raspberry Pi. Proportional feedback was used to ensure the car drove straight and adjusted turns properly.

S1XT33N

S1XT33N is a project I worked on in EECS16B, an introduction course to applied machine learning. The project involved creating a robot car named "Honey" that listens to voice commands and follows a specified path. We assembled the car from a kit and designed the circuits and code ourselves.

We built motor control circuits, regulator circuits, a microphone board, and both low and high pass filters. A linear model was created to model the noise in the signal, and we used feedback and a closed loop to ensure the car traveled straight and turned correctly. We also implemented clustering analysis to classify words as commands, all written in C using an Arduino. Commands included "honey," "uu," "dddd," and "durian."

This project has been one of the most rewarding experiences in my journey towards data science. My interest in robotics started in high school, and this class gave me the opportunity to explore it further. Through this project, I greatly improved my circuitry and debugging skills. The most challenging aspect was selecting commands that classified well in theory but were difficult to recognize when spoken. Despite these challenges, this project remains my favorite, and I am excited to apply the skills I gained in future endeavors.

Build Your Own World (BYOW)

Build Your Own World was a project for COMPSCI 61BL, a lab-based data structures and algorithms class in Java. In this project, we created a pseudorandomly generated world that changes depending on the input seed. The project included a main menu that allowed users to generate a world, load a saved world, pick an avatar, and name the avatar.

The world was composed of tiles, and we developed an engine to generate it. In each world, the avatar's name and the current date and time were displayed at the top. The world also featured teleportation tiles, event games where the player had to collect good coins and avoid bad ones, and a final puzzle that required the player to input a phrase made up of code words obtained by winning the event games. Although we couldn't implement the load feature in time, the rest of the project was completed successfully.
