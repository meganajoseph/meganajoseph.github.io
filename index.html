Projects

S1XT33N

S1XT33N is a project I worked on in EECS16B which is an introduction course to applied machine learning. The project involved making a robot car (named Honey) that listens to voice commands and follows the specified path. We used a kit that had all the parts but created the circuits and code on our own. We created motor control circuits, regulator circuits, a mic board, and a low and high pass filter. We also created a linear model to model the noise in the signal, used feedback and a closed loop to ensure the car travels in a straight line and turns properly, and used clustering analysis to classify words as commands. All programs were written in C through an Arduino. The car is powered by batteries and used the Arduino to collect data. The words we used as commands are "honey", "uu", "dddd", and "durian".

This project has been one that I looked forward to since I deciede to pursue data science. I became interested in robots in high school but never had the opportunity to explore that interest until this class. My circuitry and debugging skills have definitely improved while in this class. The hardest part of the project was picking words as commands that had a good accuracy. A majority of the time, they classified well in theory but not so much when we said the words to Honey. This project is my favorite project that I've ever completed and I am excited to use the skills I've gained and build upon them in future endeavors.

To read our midterm lab report and our final lab report, please reach out to me.


Build Your Own World (BYOW)

Build Your Own World was a project for COMPSCI 61BL, a lab-based data structures and algorithms class in Java. In this project, we made a pseudorandomly generated world that changes depending on the input seed. It contained a main menu that allowed the user to generate a world, load a saved world, pick an avatar, and name an avatar. The world was made of tiles and we created an engine to generate the world. In each world, the name of the avatar and the real date and time are displayed at the top of the viewer, there are tiles that teleport the user to random locations in the map, tiles that triggered an event game where the user had to collect all of the good coins and avoid the bad ones. Winning an event game released a code word that the user needed to remember. After clearing all the games, the user inputs the phrase made from the code words and, if it's right, wins the game. This was the most fun I had in a project because I was able to directly see how my code manifested something that I could directly interact with. We were unable to implement the load feature in time but the rest of the project is complete. One thing we wanted to implement was music and sound effects in the world. Since we had limited time to complete the project, we decided not to add it. I learned how to use StdDraw and how to use randomness in coding.

Please reach out to me to view the code!
